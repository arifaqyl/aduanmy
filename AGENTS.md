## Engineering Discipline

Apply the relevant workflow from https://github.com/tmdgusya/engineering-discipline:

1. Reproduce failures before editing and trace the root cause rather than patching symptoms.
2. Make the smallest change that preserves the established product and trust rules.
3. Add or correct a regression test for changed behavior, then run focused tests and the full suite.
4. Review runtime behavior independently of implementation notes; use browser/API evidence for user-facing changes.
5. Measure before optimizing. Remove unnecessary abstraction or AI-generated ceremony after behavior is verified.
